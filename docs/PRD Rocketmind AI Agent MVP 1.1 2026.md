Ниже представлен обновленный текст PRD для версии **MVP 1.1 **. Документ переработан с учетом исключения живых экспертов, упрощения платежного пути и фокусировки на прямой работе с ИИ-агентами через n8n.

---
### 0) Назначение

Документ фиксирует требования к упрощенной версии MVP-1.1 web-сервиса ИИ-агентов. Основной фокус: «кейсы (проекты) → чаты с ИИ-агентами» с интеграцией логики через n8n. Используется для оценки, разработки, QA и приёмки.

### 1) Рамки итерации (Scope)

- **Итерация:** MVP-1.1 (февраль-март 2026).
    
- **Канал:** Web-only.
    
- **Онбординг:** Исключен. Вход осуществляется напрямую в интерфейс чата после авторизации.
    
- **Живые эксперты:** Исключены из интерфейса сервиса. Взаимодействие ограничено только ИИ-агентами.
    
- **Платежный UI:** Отдельные экраны оплаты/эквайринга не разрабатываются. Все ссылки и триггеры на оплату реализуются текстовыми сообщениями или кнопками через логику n8n внутри диалогового окна.
    

### 2) Цель итерации

Предоставить пользователю инструмент для ведения нескольких «кейсов» (проектов), внутри которых он ведет диалоги с различными ИИ-агентами. Сервис должен обеспечивать стабильный цикл: «авторизация — выбор агента — диалог — получение результата/ссылки на оплату».

### 3) Success criteria (Приёмка)

Критический путь работает стабильно:

1. **Авторизация:** Проход через Login 2-step (email + code).
    
2. **Автобутстрап:** При входе по URL `/a/{agent_slug}` автоматически создается кейс и открывается чат с агентом.
    
3. **Диалог:** Отправка и получение сообщений от ИИ-агента через вебхуки n8n.
    
4. **Результат:** Получение итогового summary или ссылки на оплату/форму заявки в виде системного сообщения в чате.
    

### 4) Экраны и UI-структура

- **Экран 1: Лендинг сервиса.** СТА-кнопка «Попробовать».
    
- **Экран 2-3: Авторизация.** Поле ввода Email и ввод кода из письма.
    
    
- **Экран 4: Главный интерфейс.**
    
    - **Левый сайдбар:** Управление кейсами (создание, переименование, архив) и список активных агентов в кейсе.
        
    - **Центральная область:** Окно чата с ИИ-агентом.
        
- **Экран 5: Каталог агентов.** Выбор новых ИИ-агентов для добавления в кейс.
    

### 5) Функциональные требования

- **5.1 Сценарий взаимодействия:**
    
    - Пользователь авторизуется и выбирает кейс (или заходит по прямой ссылке на агента).
        
    - Ведет диалог в чат-интерфейсе.
        
    - Получает результат в виде текстовых сообщений или резюме (summary).
        
- **5.2 Интеграция n8n:**
    
    - Все сообщения пользователя уходят на вебхук n8n.
        
    - **Триггеры оплаты:** При достижении определенных условий сценария, n8n отправляет в чат системное сообщение со ссылкой на оплату.
        
- **5.3 Кейсы и изоляция:**
    
    - История диалогов и набор агентов строго изолированы внутри каждого кейса.
        
    - Возможность архивации и восстановления кейсов.
        

### 6) Not goals (Вне рамок спринта)

- **Интерфейс «Личного кабинета эксперта»:** Не реализуется.
    
- **Сложные анимации:** Исключены для ускорения сборки.
    
- **Кастомный UI воронки:** Используются стандартные компоненты дизайн-системы и библиотеки _shadcn/ui_.
    
- **Платежный шлюз:** Прямая интеграция эквайринга в интерфейс не производится.
    

### 7) Технический стек (Интеграции)

- **Frontend:** Вайпкодинг-инструмент (согласно техническому согласованию).
    
- **Backend/Logic:** n8n вебхуки для обработки сообщений, управления сессиями и выдачи рекомендаций.
    
- **Data Model:** Использование сущностей `users`, `cases`, `agents`, `conversations` и `messages`.
