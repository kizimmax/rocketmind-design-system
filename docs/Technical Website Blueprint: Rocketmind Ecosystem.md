# Техническое задание: Архитектура сайта Rocketmind v2

## 1. Общие принципы системы

* **Типология страниц:** Сайт строится на связке «Index-страница» (хаб/каталог) и «Child-страница» (детальная страница на базе шаблона).
* **Обязательная перелинковка:** Каждая продуктовая страница и страница агента **обязана** содержать блок **ИИ-агент** (интерактивный чат или виджет для начала диалога с переходом в SaaS).
* **Опциональная перелинковка:** Блок **Эксперт** добавляется на страницы продуктов, курсов и кейсов при наличии прямой связи.
* **Сквозная конверсия:** Основной CTA всех страниц — переход в основное приложение **Rocketmind SaaS**.

---

## 2. Сквозные блоки (Global Blocks)

*Эти блоки идентичны для всех страниц и управляются глобально.*

* **Header (Global):** Логотип, основная навигация (Продукты, Агенты, Академия, Кейсы), кнопка «Вход/Регистрация в SaaS».
* **Footer (Global):** Карта сайта, юридические ссылки, соцсети, контакты.
* **Widget AI-Agent:** Плавающий виджет или закрепленный блок для быстрого старта чата.

## 4. Шаблонные страницы (Child Templates)

### T1. Шаблон Продукта / Консалтинга

* **Секции:** Hero, Описание боли/решения, Список функций, Отзывы, Контакты.
* **Обязательный блок:** Чат с ИИ-агентом (контекстный для этого продукта).
* **Опциональный блок:** Карточка связанного эксперта.

### T2. Шаблон ИИ-агента

* **Секции:** Hero (Аватар агента), Возможности и навыки, Примеры промптов/диалогов, Use Cases.
* **Обязательный блок:** Полноэкранный или расширенный виджет чата «Попробовать агента».
* **CTA:** Кнопка «Запустить в Rocketmind App».

### T3. Шаблон Эксперта

* **Секции:** Профиль (Фото, Био), Достижения/Статистика, Стек технологий.
* **Обязательный блок:** Связанные продукты и курсы этого эксперта.
* **Перелинковка:** Ссылка на персонального ИИ-агента эксперта.

### T4. Шаблон Курса (Академия)

* **Секции:** Программа (модули), Стоимость, Результаты обучения.
* **Обязательный блок:** Блок преподавателей (ссылки на шаблоны экспертов).
* **CTA:** Записаться / Оплатить.

### T5. Шаблон Кейса

* **Секции:** Описание клиента, Проблема, Описание методологии, Результаты в цифрах.
* **Обязательный блок:** «Инструментарий» (ссылки на ИИ-агентов и продукты, использованные в кейсе).

### T6. Шаблон Методологии / Канваса

* **Секции:** Теоретическая часть, Инструкция по применению.
* **Уникальный блок:** Iframe (встроенный инструмент) или блок скачивания материалов.

## 5. Техническая логика фильтрации блоков галерей

Для всех страниц, содержащих «Раздел списка» (Product list, Portfolio list, и т.д.):

1. **Источник данных:** Динамическая выборка из соответствующей категории CMS.
2. **Тип отображения:** Grid (сетка карточек).
3. **Логика фильтрации:**
* На главной: `top_rated: true` или `featured: true` (ручной отбор).
* В хабах: Полный набор фильтров по тегам и категориям.
* Внутри шаблонов (блок «Похожее»): Фильтрация по совпадению тегов текущей страницы.
